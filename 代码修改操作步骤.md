# 修改代码后的 Git 操作步骤

每次修改完代码，按下面的顺序操作，把改动提交并推送到 GitHub。

## 1. 进入项目目录

```bash
cd C:/Users/18049/Desktop/niaoge/naioge
```

## 2. 查看当前修改状态

```bash
git status
```

绿色：已暂存，准备提交
红色：已修改但未暂存

## 3. 添加修改的文件

### 添加所有改动

```bash
git add .
```

### 只添加某个文件

```bash
git add 文件名
```

例如：

```bash
git add niaoge-cloud-bot/server.js
```

## 4. 提交到本地仓库

```bash
git commit -m "修改说明"
```

例如：

```bash
git commit -m "修复登录接口报错"
```

**提交说明要写得清楚**，方便以后查看这次改了什么。

## 5. 推送到 GitHub

```bash
git push
```

第一次推送新分支时才需要：

```bash
git push -u origin main
```

## 6. 确认推送成功

打开仓库地址查看：

https://github.com/xu403-star/naioge

## 常用命令速查

| 命令 | 作用 |
|---|---|
| `git status` | 查看当前修改状态 |
| `git add .` | 添加所有改动到暂存区 |
| `git commit -m "说明"` | 提交到本地仓库 |
| `git push` | 推送到远程 GitHub 仓库 |
| `git pull` | 从 GitHub 拉取最新代码 |
| `git log` | 查看提交历史 |
| `git diff` | 查看具体修改内容 |

## 注意事项

1. **不要上传 `node_modules/` 文件夹**：已加入 `.gitignore`，Git 会自动忽略。
2. **不要上传 `niaoge-cloud-bot.zip`**：压缩包也已在 `.gitignore` 中忽略。
3. **`$null` 和 `nul` 文件不要提交**：这些文件名特殊，会导致 Git 报错。
4. **提交前先检查**：`git status` 看看有没有不想上传的文件。

## 遇到问题

### 提示 `LF will be replaced by CRLF`

这是 Windows 换行符的警告，不影响使用，可以忽略。

### 推送失败，提示没有权限

重新登录 GitHub 授权：

```bash
git credential-manager reject https://github.com
git push
```

然后会重新弹出登录窗口。

### 提示 `Updates were rejected`

说明远程仓库有更新，先拉取再推送：

```bash
git pull
git push
```
